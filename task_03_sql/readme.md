# Блок 3. Анализ данных (SQL)

## Описание задачи
Написать SQL-запросы для следующих задач на диалекте PostgreSQL:
1.  **Задача: Вторая зарплата.** Дана таблица `Employee (id int PK, salary int)`. Написать SQL-запрос, который находит вторую по величине уникальную зарплату. Если её нет, запрос должен вернуть `null`.
2.  **Задача: Поиск дубликатов email.** Дана таблица `Person (id int PK, email varchar)`. Написать SQL-запрос, который находит все дублирующиеся email-адреса.
3.  **Задача: Клиенты без заказов.** Дана таблица `Customers (id int PK, name varchar)` и `Orders (id int PK, customerId int FK)`. Написать SQL-запрос, который возвращает всех клиентов, которые ни разу не делали заказов.

---

## Решения задач

### 1. Задача: Вторая зарплата
Для того чтобы при отсутствии второй зарплаты запрос гарантированно возвращал `NULL` (а не пустой набор строк), мы оборачиваем выборку в скалярный подзапрос.

```sql
SELECT (
    SELECT DISTINCT salary 
    FROM Employee 
    ORDER BY salary DESC 
    LIMIT 1 OFFSET 1
) AS SecondHighestSalary;
```

*   **Как это работает:**
    *   Вложенный запрос упорядочивает уникальные зарплаты по убыванию (`ORDER BY salary DESC`) и пропускает первую максимальную зарплату (`OFFSET 1`), забирая следующую одну строку (`LIMIT 1`).
    *   Внешний `SELECT (...)` преобразует результат в скалярную величину. Если вложенный запрос не вернет ни одной строки (например, в таблице 0 или 1 сотрудник), внешний запрос вернет `NULL`.

---

### 2. Задача: Поиск дубликатов email
Для поиска дубликатов мы группируем записи по email и фильтруем группы с количеством записей больше одной.

```sql
SELECT email
FROM Person
GROUP BY email
HAVING COUNT(email) > 1;
```

*   **Как это работает:**
    *   `GROUP BY email` объединяет строки с одинаковыми адресами.
    *   `HAVING COUNT(email) > 1` отсекает те группы, которые встречаются в таблице ровно один раз.

---

### 3. Задача: Клиенты без заказов
Есть два основных способа решения этой задачи. Ниже приведены оба варианта с точки зрения производительности в PostgreSQL.

#### Вариант А: Использование LEFT JOIN (Рекомендуемый / Читаемый)
Мы соединяем таблицы клиентов и заказов по ID клиента. Те клиенты, для которых в таблице заказов не нашлось записей, будут иметь `NULL` в полях таблицы `Orders`.

```sql
SELECT c.name AS Customers
FROM Customers c
LEFT JOIN Orders o ON c.id = o.customerId
WHERE o.id IS NULL;
```

#### Вариант Б: Использование NOT EXISTS (Наиболее производительный на больших объемах данных)
В PostgreSQL оператор `NOT EXISTS` обычно работает быстрее, так как планировщик может оптимизировать его через `Anti Join` без создания большой временной таблицы в памяти.

```sql
SELECT c.name AS Customers
FROM Customers c
WHERE NOT EXISTS (
    SELECT 1 
    FROM Orders o 
    WHERE o.customerId = c.id
);
```

*   *Примечание:* Использование `NOT IN` также возможно, но оно требует явной фильтрации `NULL` значений в подзапросе (`WHERE customerId IS NOT NULL`), иначе при наличии хотя бы одного `NULL` в `Orders.customerId` весь запрос `NOT IN` вернет пустой результат. Вариант с `NOT EXISTS` или `LEFT JOIN` лишен этого недостатка и является более безопасным.
