# Решение тестового задания: QA Engineer (Middle+/Senior) в «Умную Логистику»

Данный репозиторий содержит структурированное решение комплексного тестового задания. В соответствии с требованиями к оформлению, решение разделено по изолированным папкам со своими `README.md` файлами, описывающими логику работы.

Выполнены **все блоки задания** (включая обязательные и все три вариативных), чтобы продемонстрировать широкий инженерный кругозор.

## Структура проекта

1.  [**task_01_etrn**](file:///Users/yanpurvenes/Documents/Git/qa_engineer_test_assignment/task_01_etrn/readme.md) — Блок 1: Тестирование требований (Бизнес-процесс ЭТрН).
    *   Чек-лист сквозного BPMN-процесса (титулы Т1-Т4).
    *   Ключевые проверки формы на соответствие XSD-схемам ФНС.
    *   Негативные проверки и 2 структурированных баг-репорта.
2.  [**task_02_ai_prompting**](file:///Users/yanpurvenes/Documents/Git/qa_engineer_test_assignment/task_02_ai_prompting/readme.md) — Блок 2: Проектирование тестов (AI Prompting & Review).
    *   Пример составленного промпта для ИИ.
    *   Сырой ответ модели без изменений.
    *   Развернутое критическое ревью ответа ИИ с точки зрения реальной логистики и UX.
    *   Таблица граничных значений параметров формы расчета стоимости.
3.  [**task_03_sql**](file:///Users/yanpurvenes/Documents/Git/qa_engineer_test_assignment/task_03_sql/readme.md) — Блок 3: Анализ данных (SQL в PostgreSQL).
    *   Запрос на поиск второй уникальной зарплаты (с обработкой NULL).
    *   Запрос на поиск дублирующихся email-адресов.
    *   Запрос на выборку клиентов без заказов (LEFT JOIN и NOT EXISTS).
4.  [**task_04_api**](file:///Users/yanpurvenes/Documents/Git/qa_engineer_test_assignment/task_04_api/readme.md) — Блок 4: Тестирование API.
    *   Чек-лист для REST-контроллера `/api/v2/orders` (GET, POST, PUT, DELETE, HEAD).
    *   Детальная матрица HTTP статус-кодов по стандартам Google и Microsoft API Guidelines.
5.  [**task_05_1c**](file:///Users/yanpurvenes/Documents/Git/qa_engineer_test_assignment/task_05_1c/readme.md) — Блок 5: Базовые вопросы про 1С и обмены.
    *   Запрос 1С для периодического регистра сведений с условием непересечения границ веса.
    *   Детальное код-ревью функции `РассчитатьСтоимость()` (выявлено 6 критических ошибок/просадок производительности).
    *   Предложен исправленный вариант функции на встроенном языке 1С.
